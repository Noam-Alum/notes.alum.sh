Bash Variables

In Bash scripting, variables are used to store and manipulate data. There are various forms of variable expansion that help you transform and manage the data in different ways. Below is a guide covering some important forms of Bash variable expansions.

Assign Default Values: ${parameter:=word}

Short Explanation: This form assigns a default value to a variable if it is unset or null.

Syntax:

${parameter:=word}

Example:

var=""
echo ${var:="default_value"}  # Since var is empty, it will be assigned 'default_value'

Result:

default_value

Display Error if Null or Unset: ${parameter:?word}

Short Explanation: This form displays an error message and exits the script if the variable is null or unset. If the variable is set, it returns the value of the variable.

Syntax:

${parameter:?word}

Example:

var=""
echo ${var:?"Error: var is null or unset"}  # Will print the error message because var is empty

Result:

Error: var is null or unset

Use Alternate Value: ${parameter:+word}

Short Explanation: This form substitutes word if the variable is set and non-null. Otherwise, it substitutes nothing.

Syntax:

${parameter:+word}

Example:

var="value"
echo ${var:+"substitute"}  # Since var is set, it will print 'substitute'

Result:

substitute

Substring Expansion: ${parameter:offset}, ${parameter:offset:length}

Short Explanation: This allows you to extract a substring from a variable starting from the specified offset. The length is optional and limits the number of characters returned.

Syntax:

${parameter:offset}
${parameter:offset:length}

Example:

var="Hello, World!"
echo ${var:7}  # Starting from the 7th character to the end, prints 'World!'

Result:

World!

Remove Matching Prefix: ${parameter#word}, ${parameter##word}

Short Explanation: These forms remove the specified prefix pattern from the value of the variable. # removes the shortest match, while ## removes the longest match.

Syntax:

${parameter#word}
${parameter##word}

Example:

var="prefix_file.txt"
echo ${var#prefix_}  # Removes the shortest match, prints 'file.txt'
echo ${var##prefix_} # Removes the longest match, prints 'file.txt'

Result:

file.txt
file.txt

Remove Matching Suffix: ${parameter%word}, ${parameter%%word}

Short Explanation: These forms remove the specified suffix pattern from the value of the variable. % removes the shortest match, while %% removes the longest match.

Syntax:

${parameter%word}
${parameter%%word}

Example:

var="file.txt.suffix"
echo ${var%.*}   # Removes the shortest match for '.'
echo ${var%%.*}  # Removes the longest match for '.'

Result:

file.txt
file

Pattern Substitution: ${parameter/pattern/string}

Short Explanation: This form replaces the first occurrence of pattern with string. If pattern starts with /, all occurrences are replaced.

Syntax:

${parameter/pattern/string}

Example:

var="apple banana apple"
echo ${var/apple/orange}    # Replaces the first 'apple' with 'orange'
echo ${var//apple/orange}   # Replaces all 'apple' with 'orange'

Result:

orange banana apple
orange banana orange

Case Modification: ${parameter^}, ${parameter^^}, ${parameter,}, ${parameter,,}

Short Explanation: These forms modify the case of the characters in a variable, either converting them to uppercase or lowercase. ^ changes the first matched character, while ^^ changes all matched characters.

Syntax:

${parameter^}    # First character to uppercase
${parameter^^}   # All characters to uppercase
${parameter,}    # First character to lowercase
${parameter,,}   # All characters to lowercase

Example:

var="hello world"
echo ${var^}   # Converts first character to uppercase
echo ${var^^}  # Converts all characters to uppercase
echo ${var,}   # Converts first character to lowercase
echo ${var,,}  # Converts all characters to lowercase

Result:

Hello world
HELLO WORLD
hello world
hello world

Parameter Transformation: ${parameter@operator}

Short Explanation: This form allows you to transform or inspect a variable in various ways based on the operator used.

Syntax:

${parameter@Q}  # Quote the value
${parameter@E}  # Expand escape sequences
${parameter@P}  # Format as prompt string
${parameter@A}  # Display assignment statement
${parameter@a}  # Display attribute flags

Example:

var="Hello world"
echo ${var@Q}  # Prints 'Hello world' (quoted for reuse)
echo ${var@E}  # Expands escape sequences (if any)

Result:

'Hello world'
Hello world

Get the Length of a Variable: ${#parameter}

Short Explanation: This form returns the length of the value stored in a variable.

Syntax:

${#parameter}

Example:

var="Hello, World!"
echo ${#var}   # Returns the length of the string

Result:

13

Default Value with :-: ${parameter:-value}

Short Explanation: If the variable is unset or null, this form returns the specified default value (value). If the variable is set and not null, it returns the value of the variable.

Syntax:

${parameter:-value}

Example:

var=""
echo ${var:-"default"}  # Since var is empty, it returns 'default'

Result:

default

Check if Variable is Set: ${parameter:+value}

Short Explanation: This form checks if the variable is set and non-null. If so, it returns value, otherwise, it returns nothing.

Syntax:

${parameter:+value}

Example:

var="test"
echo ${var:+"non-null"}   # Prints 'non-null' because var is set and non-empty

Result:

non-null
